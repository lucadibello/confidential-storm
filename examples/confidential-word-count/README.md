# Confidential Word Count

A privacy-preserving word-count topology built on Confidential Storm. Jokes are loaded from an encrypted dataset, split into words inside an SGX enclave, counted with per-user contribution bounding, and aggregated into a differentially private histogram — all without the untrusted host ever seeing plaintext.

## Topology

```
RandomJokeSpout (host)
  │  emits sealed (payload, userId) from the encrypted dataset
  ▼
SpoutPreprocessingService (enclave)
  │  decrypts + re-encrypts with per-route AAD
  ▼
SplitSentenceBolt (host) → SplitSentenceService (enclave)
  │  splits body into individual words, each sealed separately
  ▼
UserContributionBoundingBolt (host) → UserContributionBoundingService (enclave)
  │  enforces C=100 max contributions per user
  ▼
DataPerturbationBolt (host) → DataPerturbationService (enclave)
  │  maintains per-key binary aggregation trees, adds calibrated noise
  ▼
HistogramAggregatorBolt (host) → HistogramAggregationService (enclave)
      merges partial histograms from parallel replicas
```

## Parameters

| Parameter | Value | Source |
|-----------|-------|--------|
| Epsilon | 8.0 | `DPConfig.EPSILON` |
| Delta | 1e-6 | `DPConfig.DELTA` |
| Max contributions / user | 100 | `DPConfig.MAX_CONTRIBUTIONS_PER_USER` |
| Per-record clamp | 1.0 | `DPConfig.PER_RECORD_CLAMP` |

## Prerequisites

- Java 17+ and Maven
- Apache Storm (for local-mode execution)
- Python 3.8+ with `cryptography` (`pip install -r tools/dataset/requirements.txt`)

## Quick Start

The encrypted dataset (`jokes.enc.json`) is **not** checked into version control because it is tied to a specific encryption scheme. You must seal it before building.

```bash
# 1. Seal the dataset (default: ChaCha20-Poly1305)
make seal-chacha20

# 2. Build everything (framework + common → enclave → host)
make build

# 3. Run locally (120 s, mock SGX)
make run-local
```

## Encryption Scheme

The dataset must be sealed with the **same scheme** configured in the enclave. These two must always match:

| What | Where |
|------|-------|
| Dataset sealing | `make seal-<scheme>` (or `make seal-dataset ENCRYPTION_SCHEME=...`) |
| Enclave runtime | `WordCountEnclaveConfigProvider.getEncryptionScheme()` |

### Available schemes

| Scheme | Make target | Description |
|--------|-------------|-------------|
| `CHACHA20_POLY1305` | `make seal-chacha20` | Default. Software-friendly AEAD cipher. |
| `AES_256_GCM` | `make seal-aes256gcm` | Hardware-accelerated on Intel/AMD (AES-NI). |
| `NONE` | `make seal-none` | Plaintext passthrough for benchmarking. No confidentiality. |

### Switching scheme

1. Seal the dataset with the desired scheme:
   ```bash
   make seal-aes256gcm
   ```

2. Update `WordCountEnclaveConfigProvider` to return the matching scheme:
   ```java
   @Override
   public EncryptionScheme getEncryptionScheme() {
       return EncryptionScheme.AES_256_GCM;
   }
   ```

3. Rebuild and run:
   ```bash
   make build
   make run-local
   ```

If the dataset scheme and the enclave scheme do not match, the topology will crash with a `SealedPayloadProcessingException` at the first decryption attempt.

## Project Structure

```
confidential-word-count/
├── common/          # Shared API interfaces and DP config
├── enclave/         # SGX enclave service implementations
├── host/            # Storm topology, bolts, and spout
│   └── src/main/resources/
│       └── jokes.enc.json   ← generated by make seal-*
├── tools/
│   ├── dataset/
│   │   ├── jokes.json            # Raw plaintext jokes (checked in)
│   │   ├── regen-dataset/        # Adds synthetic user_id annotations
│   │   ├── seal-dataset/         # Encrypts dataset with selected scheme
│   │   └── stream_key_hex        # Demo symmetric key (32 bytes, hex)
│   └── max-contribution-estimation/
└── Makefile
```

## Make Targets

Run `make help` for the full list. Key targets:

| Target | Description |
|--------|-------------|
| `make build` | Clean + build common, enclave (native), host |
| `make run-local` | Run topology on a local Storm cluster (120 s) |
| `make seal-chacha20` | Seal dataset with ChaCha20-Poly1305 |
| `make seal-aes256gcm` | Seal dataset with AES-256-GCM |
| `make seal-none` | Seal dataset with no encryption (benchmarking) |
| `make seal-dataset ENCRYPTION_SCHEME=X` | Seal with an explicit scheme |
| `make test-enclave` | Run enclave unit tests |
| `make percentile` | Compute per-user word-count percentile (p=0.99) |
| `make submit` | Submit topology to a remote Storm cluster |
